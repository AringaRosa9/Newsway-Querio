## AI News Search とは？

ジャーナリスト、アナリスト、研究者向けの **AI 駆動型ニュース検索エンジン**です。従来のニュース検索とは異なり、以下の機能を提供します：

- **自然言語クエリ** — キーワードを組み合わせる必要なし、日常の言葉で直接質問
- **AI 統合要約** — 複数の報道から総合的な回答を生成し、出典を明記
- **イベントタイムライン** — 同一イベントの報道を自動集約し、時系列で展開
- **リアルタイム通知** — キーワードやイベントを購読し、実質的な新展開があれば即時通知

| 側面 | 従来のニュース検索 | AI News Search |
|---|---|---|
| 入力方式 | キーワード | 自然言語での質問 |
| 結果形態 | リンクの一覧 | AI 要約 + 出典リンク |
| 情報整理 | 記事ごとの羅列 | イベントベースの集約 + タイムライン |
| 継続追跡 | 繰り返しの手動検索 | 購読して自動通知 |


## 主要機能

### インテリジェント検索

自然言語で入力すると、システムが自動的に意図を識別しクエリを書き換えます。中国語・英語の混合入力に対応。

- **クエリ理解**：キーエンティティ、時間範囲、検索意図を抽出
- **ハイブリッド検索**：BM25 キーワード + BGE-M3 セマンティックベクトルの二重リコール、RRF で統合
- **AI リランキング**：関連性(40%) + 鮮度(25%) + 権威性(20%) + 多様性(15%) のマルチファクターランキング

### AI 要約

検索結果の Top-10 文書から総合的な回答を生成し、出典リンクを付与します。

- 多角的な提示：事実の概要、各方面の見解、発展の傾向
- 情報の追跡：各要約に出典を明記、クリック可能なリンク付き
- ハルシネーション制御：事実正確率目標 ≥ 95%

### イベント追跡

同一イベントに関する複数の報道を自動的にクラスタリングし、タイムラインを生成します。

- セマンティッククラスタリングで同一イベントの異なる報道を識別
- 時系列のタイムラインでイベントの進展を表示
- キーエンティティ（人物、組織、場所）とセンチメント推移を注記

### パーソナライズ通知

キーワード、トピック、またはイベントを購読し、実質的な新展開があれば通知を受け取れます。

- メールダイジェスト（日次/週次）とリアルタイムプッシュ通知の2モード対応
- 重複・非実質的なコンテンツのスマートフィルタリング
- 閲読履歴に基づく関心プロファイル構築、パーソナライズされたランキング

## 技術スタック

| レイヤー | 技術 | 用途 |
|---|---|---|
| フロントエンド | **Next.js** | SSR 検索ページ、SEO 最適化 |
| バックエンド | **FastAPI (Python)** | API サービス、ビジネスロジック |
| キーワード検索 | **Elasticsearch 8.x** | BM25 全文検索 |
| ベクトル検索 | **Qdrant** | セマンティックベクトル格納 & ANN 検索 |
| Embedding | **BGE-M3** | 多言語テキストのベクトル化 |
| リランキング | **BGE-Reranker-v2** | 検索結果の精密ランキング |
| LLM | **Claude API** | 要約生成、対話、クエリ理解 |
| データ取得 | **Scrapy + RSSHub** | RSS/API ニュース取得 |
| キャッシュ/キュー | **Redis** | 人気クエリのキャッシュ、タスクキュー |
| モニタリング | **Grafana + Prometheus** | システム監視、評価ダッシュボード |

## クイックスタート

### 前提条件

- Python 3.11+
- Node.js 20+
- Docker & Docker Compose

### 1. リポジトリのクローン

```bash
git clone <repo-url>
cd ai-news-search
```

### 2. インフラの起動

```bash
# Elasticsearch + Qdrant + Redis を一括起動
docker compose up -d
```

### 3. 環境変数の設定

```bash
cp .env.example .env
# .env を編集し、Anthropic API Key を入力
```

### 4. バックエンドの起動

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload
```

### 5. フロントエンドの起動

```bash
cd frontend
npm install
npm run dev
```

`http://localhost:3000` にアクセスして利用を開始できます。

## 評価体系

本プロジェクトはオフライン・オンライン・人的評価の三層フレームワークを構築し、検索品質を継続的に監視します。

### オフライン評価（リリース前）

| 指標 | 目標値 | 説明 |
|---|---|---|
| NDCG@10 | ≥ 0.75 | Top-10 結果のランキング品質 |
| Recall@50 | ≥ 0.85 | 関連文書の再現率 |
| MAP | ≥ 0.70 | 平均精度 |
| MRR | ≥ 0.60 | 最初の関連結果のランク位置 |
| 事実正確率 | ≥ 95% | ハルシネーションのない AI 要約の割合 |
| ROUGE-L | ≥ 0.45 | 要約と参照テキストの重複度 |

### オンライン評価（リリース後）

| 指標 | 目標値 | 説明 |
|---|---|---|
| 検索成功率 | ≥ 70% | クリックが発生した検索の割合 |
| ゼロ結果率 | ≤ 5% | 結果が返されないクエリの割合 |
| 再検索率 | ≤ 20% | ユーザーがクエリを再構成するセッションの割合 |
| P95 レイテンシー | ≤ 3s | エンドツーエンドの検索応答時間 |
| NPS | ≥ 40 | ネットプロモータースコア |
