# 开发计划 / Development Plan

> **预计周期**：16 周（可根据团队规模调整）  
> **更新日期**：2026-07-21  

---

## Phase 0 · MVP（第 1–6 周）

目标：跑通"搜索 → 召回 → 排序 → AI 摘要 → 展示"核心链路，内部可用。

### Sprint 1（Week 1–2）· 基础设施 + 数据管道

| # | 任务 | 负责角色 | 产出物 | 完成标准 |
|---|---|---|---|---|
| 1.1 | 项目脚手架：后端 FastAPI + 前端 Next.js + Monorepo 配置 | 全栈 | 可运行的空项目 | `npm run dev` / `uvicorn` 启动无报错 |
| 1.2 | Docker Compose 编排：ES 8.x + Qdrant + Redis | 后端 | docker-compose.yml | `docker compose up` 一键拉起全部依赖 |
| 1.3 | CI/CD Pipeline（GitHub Actions） | 后端 | lint + test + build 流水线 | PR 合并前自动检查通过 |
| 1.4 | RSS/API 采集器：支持 20+ 新闻源 | 后端 | ingestion 模块 | 每 15 分钟自动采集，数据写入 ES |
| 1.5 | 数据清洗：正文提取 + 去广告 + 字段标准化 | 后端 | cleaning pipeline | 正文提取准确率 ≥ 95%（人工抽查 100 篇） |

**里程碑 M1（Week 2 末）**：混合检索 Demo 可运行，50 个测试 query NDCG@10 ≥ 0.65。

### Sprint 2（Week 3–4）· 搜索引擎 + AI 处理

| # | 任务 | 负责角色 | 产出物 | 完成标准 |
|---|---|---|---|---|
| 2.1 | NLP 处理管道：分类 + 实体抽取 + 情感分析 | 算法 | ai/nlp 模块 | 分类准确率 ≥ 85% |
| 2.2 | Embedding 向量化：BGE-M3 接入 + 批量入库 | 算法 | ai/embedding 模块 | 全量新闻向量化，入库 Qdrant |
| 2.3 | 混合检索：BM25 + 向量双路召回 + RRF 融合 | 后端 | search/retrieval 模块 | Recall@50 ≥ 0.80 |
| 2.4 | AI Re-Ranking：相关性 + 时效性 + 权威性多因子排序 | 算法 | search/ranking 模块 | NDCG@10 ≥ 0.70 |
| 2.5 | 去重管道：SimHash + 语义相似度双重去重 | 后端 | ingestion/dedup 模块 | 重复文章检出率 ≥ 95% |

### Sprint 3（Week 5–6）· AI 摘要 + 前端 + 评估基线

| # | 任务 | 负责角色 | 产出物 | 完成标准 |
|---|---|---|---|---|
| 3.1 | Query 理解：意图识别 + Query 改写 + 时间推断 | 算法 | search/query 模块 | 意图识别准确率 ≥ 90% |
| 3.2 | AI 摘要生成：基于 Top-10 文档综合回答 + 引用溯源 | 算法 | ai/summary 模块 | 事实准确率 ≥ 95%，含引用链接 |
| 3.3 | 搜索页 UI：搜索框 + 摘要卡片 + 结果列表 + 筛选面板 | 前端 | 搜索页面 | 完整搜索流程可走通 |
| 3.4 | 离线评估 Pipeline：标注数据集 + 自动跑批 + 报告生成 | 后端 | evaluation 模块 | 500+ 标注对，评估报告自动输出 |
| 3.5 | 基础运维：日志收集 + 错误告警 + 健康检查 | 后端 | 监控配置 | Grafana 看板可查看系统状态 |

**里程碑 M2（Week 6 末）**：MVP 上线，内部用户可完成完整搜索流程。
